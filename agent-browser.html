agent-browser cheatsheet

Fast browser automation CLI for AI agents. Drives Chrome via CDP with stable element refs.

Homeagent-browser.dev Installbrew install agent-browser Thenagent-browser install
Loop: open → snapshot -i (get @e1, @e2 refs) → click/fill/press → wait → re-snapshot. Refs invalidate after navigation or DOM change.

brew install agent-browser npm install -g agent-browser cargo install agent-browser agent-browser install agent-browser install --with-deps agent-browser --version agent-browser doctor --fix agent-browser upgrade

Screenshots (tips for vision models)

agent-browser screenshot # viewport, auto-named in temp dir agent-browser screenshot ./shot.png # explicit path agent-browser screenshot --full ./page.png # full scrolling page agent-browser screenshot --annotate # overlays [N] labels + legend agent-browser screenshot --annotate ./a.png # save annotated agent-browser screenshot ".modal" ./modal.png # scope to selector (element shot) agent-browser screenshot --screenshot-format jpeg --screenshot-quality 75 ./s.jpg agent-browser screenshot --screenshot-dir ./shots agent-browser screenshot --annotate --json # JSON legend for agent consumption
  • Annotate → ref: --annotate labels [1], [2]… map 1:1 to @e1, @e2 from snapshot -i. Pass the image + legend to a vision model, get back a label, then click @eN.
  • Full page vs viewport: --full stitches the whole scrollable page — use it to capture below-the-fold content. Otherwise only current viewport.
  • Element shot: pass a selector before the path to crop to one element — cheaper tokens than the full page.
  • JPEG for long pages: --screenshot-format jpeg --screenshot-quality 70-85 cuts file size >10× vs PNG. Use PNG only when text sharpness matters.
  • Settle first: screenshots freeze the current frame. Call wait 500 or wait @eN after click/open so animations/lazy content land before capture.
  • Dark mode: --color-scheme dark on the session flag (not screenshot) forces prefers-color-scheme.
  • Viewport size: agent-browser set viewport 1440 900 before the shot — controls layout + resolution of the output.
  • Default dir: set AGENT_BROWSER_SCREENSHOT_DIR=./shots in env so you don’t pass paths each call.

Create a Claude skill for agent-browser

The CLI ships skill content via agent-browser skills. Wire it into Claude Code by dropping a SKILL.md at ~/.claude/skills/agent-browser/SKILL.md — Claude loads it on session start.

agent-browser skills list # all built-in skills agent-browser skills get core # overview + common patterns agent-browser skills get core --full # full command reference agent-browser skills get electron # specialized: electron apps agent-browser skills get slack # specialized: slack agent-browser skills path core # print directory of skill files

One-liner install into Claude Code:

mkdir -p ~/.claude/skills/agent-browser
agent-browser skills get core --full > ~/.claude/skills/agent-browser/SKILL.md

SKILL.md frontmatter (add at top if not already present — required by Claude Code):

---
name: agent-browser
description: Browser automation CLI for AI agents. Use when the user needs
  to navigate pages, fill forms, click buttons, take screenshots, extract
  data, or automate any browser task.
allowed-tools: Bash(agent-browser:*), Bash(npx agent-browser:*)
---
  • Scope: user-wide at ~/.claude/skills/, or project-scoped at <repo>/.claude/skills/. Project overrides user.
  • Refresh: after brew upgrade agent-browser, re-run skills get core --full > SKILL.md — skills are version-matched to the CLI.
  • Discover: the find-skills skill in Claude Code can auto-suggest and install agent skills on demand.
  • Verify: / in Claude Code then check the skill list, or look for agent-browser in the session’s available-skills block.
  • Custom skill: copy ~/.agent-browser/skills/core/ as a starting template and edit for your workflow (login helpers, site-specific selectors).

Batch & chaining

agent-browser open x.com \ && agent-browser snapshot -i \ && agent-browser screenshot agent-browser batch \ "open x.com" \ "fill @e1 hello" \ "click @e2" agent-browser batch --bail ... # stop on err

Daemon keeps the browser alive between calls — shell && chains reuse state without relaunching Chrome.
