docker cheatsheet

Containers, images, volumes, networks, and compose

Homedocker.com Installdocs.docker.com/get-docker Docsdocs.docker.com
Mental model: image = template, container = running instance. docker run = pull + create + start. Most subcommands accept <name> or first few chars of <id>.

Images

docker pull image:tag docker images docker image ls --filter dangling=true docker rmi image:tag docker image prune docker image prune -a docker build -t name:tag . docker build --no-cache -t n:t . docker tag src:tag registry/n:tag docker push registry/n:tag docker save img:tag > img.tar docker load < img.tar docker history image:tag

Run containers

docker run image docker run -it image bash docker run -d image # detached docker run --rm image # auto-clean docker run --name app image docker run -p 8080:80 image docker run -v $PWD:/app image docker run -v data:/var image docker run -e KEY=val image docker run --env-file .env image docker run --network=host image docker run --restart=unless-stopped docker run -u 1000:1000 image docker run --cpus=2 --memory=1g

Manage containers

docker ps docker ps -a docker stop name docker start name docker restart name docker kill name # SIGKILL docker rm name docker rm -f $(docker ps -aq) docker logs name docker logs -f --tail 100 name docker exec -it name bash docker inspect name docker stats docker top name docker cp file name:/path docker cp name:/path file docker commit name img:tag docker diff name # fs changes

Volumes

docker volume create mydata docker volume ls docker volume inspect mydata docker volume rm mydata docker volume prune # bind vs named volume -v $PWD:/app # bind mount -v mydata:/data # named volume -v /data # anonymous

Networks

docker network ls docker network create mynet docker network create \ --driver bridge \ --subnet 172.28.0.0/16 mynet docker network inspect mynet docker network connect mynet name docker network disconnect mynet name docker network rm mynet docker network prune

System

docker info docker version docker system df docker system prune # safe-ish docker system prune -a --volumes docker login docker logout docker context ls docker context use myctx

Dockerfile essentials

FROM node:20-alpine AS build WORKDIR /app COPY package*.json ./ RUN npm ci COPY . . RUN npm run build FROM nginx:alpine COPY --from=build /app/dist \ /usr/share/nginx/html ENV NODE_ENV=production ARG VERSION=dev EXPOSE 80 USER 1000 HEALTHCHECK --interval=30s \ CMD curl -f http://localhost/ || exit 1 ENTRYPOINT ["nginx"] CMD ["-g", "daemon off;"]

Compose

docker compose up docker compose up -d docker compose up --build docker compose down docker compose down -v # w/ volumes docker compose ps docker compose logs -f docker compose logs -f service docker compose exec service bash docker compose run --rm service cmd docker compose build docker compose build --no-cache docker compose pull docker compose restart service docker compose config # validate docker compose -f prod.yml up

Debug & inspect

# what's eating disk docker system df -v # container exit code & reason docker inspect name \ --format='{{.State.ExitCode}} \ {{.State.Error}}' # get IP of running container docker inspect name \ --format='{{.NetworkSettings.IPAddress}}' # stream events docker events # dump full config as JSON docker inspect name | less

Common patterns

# kill everything docker rm -f $(docker ps -aq) # nuke images docker rmi $(docker images -q) # shell into running prod container docker exec -it $(docker ps \ -qf name=api) sh # one-off container, auto-clean docker run --rm -it alpine sh # tail all compose logs docker compose logs -f --tail=50
Tip: docker system prune -a --volumes reclaims the most disk — but also deletes unused images / volumes. Read before running on shared machines.
