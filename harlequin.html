harlequin cheatsheet

TUI SQL IDE for DuckDB, Postgres, SQLite, MySQL, BigQuery, Databricks, and more.

Homeharlequin.sh Repotconbeer/harlequin Installbrew install harlequin · uv tool install harlequin
Default keymap: VS Code-style. Run query Ctrl+Enter. Cycle focus F2 editor ↔ F5 results ↔ F6 catalog. Help F1.

Install

uv tool install harlequin pipx install harlequin pip install harlequin brew install harlequin # with adapter extras: uv tool install 'harlequin[postgres]' uv tool install 'harlequin[postgres,mysql,s3]' # community adapter (separate pkg): uv tool install harlequin-snowflake

Python 3.9+. DuckDB and SQLite ship built-in — no extra needed.

Adapters (extras)

postgres mysql bigquery databricks trino odbc adbc cassandra nebulagraph s3

Launch

harlequin # DuckDB in-mem harlequin path/to/duck.db # DuckDB file harlequin -a sqlite db.sqlite harlequin -a postgres \ "postgres://user:pw@host:5432/db" harlequin -a mysql --host h \ --user u --database d harlequin -a bigquery \ --project my-project harlequin -a databricks \ --server-hostname H --http-path P \ --access-token T harlequin --profile prod-pg harlequin --help # adapter opts

Config & profiles

Path: ~/.config/harlequin/config.toml (XDG). Also reads ~/.harlequin.toml, ./.harlequin.toml, ./pyproject.toml ([tool.harlequin]). Override with --config-path. Generate interactively: harlequin --config. Live key dump: harlequin --keys.

default_profile = "local-duck"

[profiles.local-duck]
adapter = "duckdb"
conn_str = ["./my.db"]
theme = "monokai"

[profiles.prod-pg]
adapter = "postgres"
host = "db.example.com"
port = 5432
user = "me"
database = "app"

# layered keymaps: vscode base + custom overrides
keymap_name = ["vscode", "my_keys"]

[[keymaps]]
name = "my_keys"
bindings = [
  { keys = "ctrl+t", action = "new_buffer" },
]

Avoid zellij / tmux Ctrl conflicts

Inside a zellij or tmux pane, several harlequin Ctrl bindings get swallowed by the multiplexer before reaching the TUI. Worst offenders: Ctrl+Q (zellij quits the whole session), Ctrl+S (zellij scroll mode), Ctrl+N / Ctrl+O (zellij resize / session modes). Layer an Alt-prefix keymap on top of vscode to dodge the collisions:

keymap_name = ["vscode", "no-mux-conflict"]

[[keymaps]]
name = "no-mux-conflict"
bindings = [
  { keys = "alt+q", action = "quit",          key_display = "Alt+Q" },
  { keys = "alt+s", action = "save_buffer",   key_display = "Alt+S" },
  { keys = "alt+o", action = "load_buffer",   key_display = "Alt+O" },
  { keys = "alt+n", action = "new_buffer",    key_display = "Alt+N" },
  { keys = "alt+w", action = "close_buffer",  key_display = "Alt+W" },
  { keys = "alt+e", action = "export",        key_display = "Alt+E" },
  { keys = "alt+r", action = "refresh_catalog", key_display = "Alt+R" },
  { keys = "alt+b", action = "toggle_sidebar", key_display = "Alt+B" },
]

Mnemonic stays identical — only swap Ctrl for Alt (Option on macOS; enable “Use Option as Meta” in Terminal/iTerm/Ghostty). Ctrl+Enter for run query is safe — multiplexers don’t bind it. Verify your live map: harlequin --keys.

Tip: harlequin --limit 1000 caps result rows. Ctrl+E exports CSV / Parquet / JSON via DuckDB engine even when querying remote DBs.
